Teddy – introduction

Teddy is a tiled text editor inspired by the Acme text editor but more configurable and more keyboard friendly.

In addition to editing files you can execute external commands and see their output from inside the editor.

Teddy supports a scripting language to write editing macros, TAGS files produced by the command ctags, syntax highlighting and autocompletion.

Fundamental concepts

Frames

Each teddy window is divided into one or more columns, each column is divided into one or more editing frames. Editing frames (in short "frames") either show the editable contents of a file, the output of a command or the contents of a directory.

The top line contains the current directory and the "Tools" menu. When <Esc> is pressed the top line is replaced with a command line, typing a command here and pressing enter will execute it.

You can create a new column by choosing the menu item "New column" from the "Tools" menu. New frames are created as needed when opening files, directories or executing commands. To open a file or a directory specifying its full path press <Esc>: key then type buffer open path-to-file, by picking "Search file" from "Tools" menu you can search for a file recursively.

Elements of a frame

  • Resize Handle: When dragged with the mouse will resize the frame (and the column containing the frame). Its color also shows if the currently edited buffer was modified after the last save or not. When it's white (like in the image) the buffer was not modified, when it's blue the buffer was modified and needs to be saved.
  • Tag: Shows the name of the current buffer. Some mouse operations are available on this GUI element.
  • Editing Area: Lets the user edit the buffer associated with this frame.
  • Position Box: Shows the current position of the cursor in the editing area, both as absolute position (line number, glyph number) and as percentage of the complete file.

Output buffers and links

By pressing <Esc>, typing an external command (for example make) and pressing >Enter< the external command will be executed and its output displayed on an output frame. When teddy finds names of files in the output of an external command it will transform them into hyperlinks. In the example, clicking "editor.c:17:18" will open the file editor.c (or focus its frame if it's already open) and go to the 18th column of the 17th line.

Default keybindings

When teddy is first started a configuration file is created in $HOME/.config/teddy/rc. You are encouraged to read that file.

Interaction

There are three ways to interact with teddy:

  • Using the mouse
  • Using the keyboard
  • Using the command line
